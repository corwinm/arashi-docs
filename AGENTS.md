# Arashi Docs Agent Rules

This repository contains the Arashi documentation site.

## Scope

- Put site content in `docs/`.
- Put site configuration and tooling changes in the repo root and `scripts/` as needed.
- Keep docs pages focused on user outcomes and link to command references instead of duplicating behavior everywhere.

## Working Rules

- Keep the landing page concise.
- Prefer clear workflow guidance over long command dumps.
- Preserve existing Starlight and Astro conventions.

## Validation

- `bun run validate`
